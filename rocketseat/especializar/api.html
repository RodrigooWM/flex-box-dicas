O Objetivo do JSON é a troca de dados entre sistemas diferentes.
Fique atento, pois existe um formato a ser seguido, por padrão, na criação de um documento json. Os elementos são separados por vírgulas, podemos inserir classes e subclasses diferenciando-as com chaves e colchetes. Identificamos através de áspas esses elementos, etc.

Verbos HTTP:
Get → Busca as informações do servidor, que são fornecidas por meio de uma API.
POST → Recebe as informações que devem ou não serem registradas.
PUT → Atualiza as informações de um ou mais registros.
PATCH → Permite atualizar informações de registro por partes.
DELETE → Apaga informações através de um dado do registro que possa identificá-lo.
